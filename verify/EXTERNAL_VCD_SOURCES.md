# External VCD sample sources

The bundled external samples are small files downloaded from public GitHub raw URLs.
Use them as smoke tests only; the synthetic unit tests cover parser corner cases more systematically.

Included samples:

1. `external_sample_GordonMcGregor_sample.vcd`
   - Source: `https://raw.githubusercontent.com/GordonMcGregor/vcd_parser/master/sample.vcd`
   - Useful coverage: classic single-line VCD, trireg/reg/integer declarations, dumpvars/dumpall/dumpoff/dumpon.

2. `external_sample_myhdl_simple_memory.vcd`
   - Source: `https://raw.githubusercontent.com/aibtw/myHdl_Projects/main/SimpleMemory/Simple_Memory.vcd`
   - Useful coverage: MyHDL output, aliases/shared identifier codes across hierarchy, memory-style names `mem(0)`, many 8-bit vectors.

3. `external_sample_vcs_empty_dump.vcd`
   - Source: `https://raw.githubusercontent.com/Mhd-Shah/Verification-of-I2C-communication-protocol/main/dump.vcd`
   - Useful coverage: Synopsys VCS header metadata and an empty `$dumpvars` data section.

Other candidates not bundled because they are snippets, issue text, or generated artifacts rather than stable small raw files:

- `https://github.com/nakane1chome/cpp-vcd-tracer` documents a VCD with real values and mixed digital/analog signals.
- `https://github.com/antmicro/usb-test-suite-testbenches` mentions VCD traces generated by the test suite; these are likely generated outputs, not small stable checked-in fixtures.
- `https://github.com/gtkwave/gtkwave/issues/460` contains string value-change examples; current parser intentionally does not support VCD string value changes.

If you want to add larger real traces locally, set `VCD_EXTERNAL_DIR=/path/to/vcds` and add a pytest smoke test that iterates over `*.vcd` in that folder. Keep large traces out of the committed test package.
