# v1.0.0

## Release Summary

`v1.0.0` is the first archived public snapshot of `vcd_analyzer`.
There is no prior version to diff against, so this note is based on direct code inspection of [vcd_analyzer_1_0_0.py](https://github.com/neveltyc/VCD_ANALYZER/blob/v1.0.0/vcd_analyzer.py) and the current sanitized regression coverage written specifically for this tag.

## Feature Set

### Core CLI surface

This version ships the following subcommands:

- `info`
- `list`
- `dump`
- `summary`
- `edges`
- `handshake`
- `snapshot`
- `compare`
- `search`

Code evidence:

- command registration is in [vcd_analyzer_1_0_0.py](https://github.com/neveltyc/VCD_ANALYZER/blob/v1.0.0/vcd_analyzer.py)
- command implementations span [vcd_analyzer_1_0_0.py](https://github.com/neveltyc/VCD_ANALYZER/blob/v1.0.0/vcd_analyzer.py) through [vcd_analyzer_1_0_0.py](https://github.com/neveltyc/VCD_ANALYZER/blob/v1.0.0/vcd_analyzer.py)

### Supported options in this version

- Global: `--json`, `--version`
- Common filtering: `--filter`
- Time windowing: `--begin`, `--end`, `--at`
- Legacy search interface: `--signal`, `--value`

### Functional behavior introduced in v1.0.0

- Streams VCD data from disk and reconstructs hierarchical signal paths from `$scope` / `$var`.
- Formats scalar, vector, event, and extended-VCD port values for RTL-oriented inspection.
- Provides point-in-time waveform inspection with `snapshot` and two-point diffs with `compare`.
- Provides clock-oriented edge counting and crude frequency estimation through `edges`.
- Auto-discovers AXI-stream style channels using `tvalid` / `tready` / `tdata` / `tlast` suffixes in `handshake`.
- Supports a legacy point-search interface through `search --signal ... --value ...`.

## Behavior Contracts And Known Limitations

These are not guesses; they come directly from the `v1.0.0` implementation and therefore should be preserved by tests for this tag.

### `dump` ignores `--json`

`cmd_dump()` in `v1.0.0` always prints text and never branches on `args.json`.  
This is important because later versions do provide structured JSON here. The `v1.0.0` test suite now asserts the real historical behavior instead of forcing later semantics backward.

Code evidence:

- [vcd_analyzer_1_0_0.py](https://github.com/neveltyc/VCD_ANALYZER/blob/v1.0.0/vcd_analyzer.py)

### `summary` counts the initial observation as a change

`cmd_summary()` initializes each signal's `changes` counter from the first event it sees in the selected window, including `t=0` initialization.  
Later versions refine summary semantics, but `v1.0.0` tests must lock in the original behavior.

Code evidence:

- [vcd_analyzer_1_0_0.py](https://github.com/neveltyc/VCD_ANALYZER/blob/v1.0.0/vcd_analyzer.py)

### `search` returns point matches, not intervals

The initial search command scans events and returns individual hits matching `--value`; it does not compute stable intervals.  
This is a fundamental interface distinction versus the later `1.1.8+` interval-style legacy search and the `1.3.x` condition-based search redesign.

Code evidence:

- [vcd_analyzer_1_0_0.py](https://github.com/neveltyc/VCD_ANALYZER/blob/v1.0.0/vcd_analyzer.py)

### `handshake` depends on naming convention discovery

`handshake` does not take explicit signal names. It only works when discovered leaf names end in `tvalid`, `tready`, `tdata`, or `tlast`.  
If no group is found, the command prints a text diagnostic instead of returning structured JSON.

Code evidence:

- [vcd_analyzer_1_0_0.py](https://github.com/neveltyc/VCD_ANALYZER/blob/v1.0.0/vcd_analyzer.py)

## Test Design For v1.0.0

The current `v1.0.0` regression set is written specifically around this tag's historical behavior, not around later APIs.

### Command coverage

Each command is executed at least once:

- `info`
- `list`
- `dump`
- `summary`
- `edges`
- `handshake`
- `snapshot`
- `compare`
- `search`

### Option coverage

The following options are exercised in `v1.0.0` tests:

- `--version`
- `--json`
- `--filter`
- `--begin`
- `--end`
- `--at`
- `--signal`
- `--value`

### Fixture strategy

- `basic_trace.vcd` covers generic waveform browsing, summary, compare, dump, and legacy search.
- `handshake_trace.vcd` uses sanitized AXI-like naming (`lane_tvalid`, `lane_tready`, `lane_tdata`) to test auto-discovery.
- `search_trace.vcd` is kept for later tags but is harmless in this versioned test set.
- `bus_range_trace.vcd` and `escaped_trace.vcd` are future regression fixtures; they are present in the repo for consistency but are not asserted in `v1.0.0`.
