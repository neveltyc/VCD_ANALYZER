# v1.1.0

## Diff Base

This note is based on the code delta from:

- [v1.0.0](../archive/vcd_analyzer_1_0_0.py)

The source diff is not a cosmetic edit. It removes one command and rewrites a large part of the parser around token-based header/data handling.

## Feature Changes

### Removed command

`handshake` was removed from the public CLI in `v1.1.0`.

Code evidence:

- present in [v1.0.0](../archive/vcd_analyzer_1_0_0.py)
- absent in [v1.1.0](../archive/vcd_analyzer_1_1_0.py)

Impact:

- this is a backward-incompatible CLI change
- any release note or release title for `v1.1.0` must call this out explicitly
- historical tests for `v1.1.0` must no longer expect `handshake` to exist

### Retained command surface

The following commands remain available:

- `info`
- `list`
- `dump`
- `summary`
- `edges`
- `snapshot`
- `compare`
- `search`

## Parser And Behavior Changes

### Token-based header parsing

`v1.1.0` replaces the line-oriented header parser with a token-driven parser using declaration keyword tracking.

Code evidence:

- new declaration keyword table in [v1.1.0](../archive/vcd_analyzer_1_1_0.py)
- new token-based `_parse_header()` structure in [v1.1.0](../archive/vcd_analyzer_1_1_0.py)

Why this matters:

- multi-line `$timescale`, `$scope`, `$var`, and `$enddefinitions` bodies are handled more systematically
- parser behavior is less tied to original line breaks

### Support for inline simulation keyword blocks

`v1.1.0` introduces explicit simulation block keyword handling with `_SIM_KEYWORDS` and a token stream over the data section.

Code evidence:

- `_SIM_KEYWORDS` in [v1.1.0](../archive/vcd_analyzer_1_1_0.py)
- `_data_tokens()` in [v1.1.0](../archive/vcd_analyzer_1_1_0.py)
- token-based `iter_events()` in [v1.1.0](../archive/vcd_analyzer_1_1_0.py)

Probable bug fix intent:

- allow `$dumpvars`, `$dumpall`, `$dumpon`, `$dumpoff` bodies to be parsed correctly even when they are inline, multi-line, or wrapped awkwardly
- preserve data tokens buffered on the same line as `$enddefinitions $end`

### Signal metadata becomes richer

`signals` entries are widened from `{path, width}` to `{path, width, type, aliases}`.

Code evidence:

- old shape comment in [v1.0.0](../archive/vcd_analyzer_1_0_0.py)
- new shape comment in [v1.1.0](../archive/vcd_analyzer_1_1_0.py)

This is not just internal cleanup. It improves alias preservation and type-aware formatting downstream.

## What Did Not Change Yet

These behaviors still belong to the `1.1.0` contract:

- `dump` still does not emit structured JSON even if `--json` is passed
- `search` is still the legacy `--signal/--value` interface
- `edges` still exists

Those points matter because later versions change them, and the regression suite for `v1.1.0` must remain historically accurate.
