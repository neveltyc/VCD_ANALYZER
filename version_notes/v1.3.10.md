# v1.3.10

## Diff Base

This note is based on the code delta from:

- [v1.3.9](https://github.com/neveltyc/VCD_ANALYZER/blob/v1.3.9/vcd_analyzer.py)
- [v1.3.10](https://github.com/neveltyc/VCD_ANALYZER/blob/v1.3.10/vcd_analyzer.py)

## Release Headline

Fix `summary` begin-boundary transition counting

## Feature / Fix Summary

- Fix the boundary black-hole in `summary` where value_changes exactly at `--begin` (with `--begin > 0`) were absorbed into baseline and silently dropped from changes / rise_count / fall_count / first_at.
- Preserve `v1.3.9` behaviour for `--begin 0`: t=0 $dumpvars initialization events establish baseline state and are not counted as changes.
- Keep `search`, `snapshot`, `compare`, `dump`, parser behaviour, and filter behaviour unchanged.

## CLI Surface Changes

No CLI subcommands or flags were added or removed in this version.

## Code-Level Change Shapes

Modified helpers:

- `_summary_rows(...)`
  - baseline capture now uses a conditional `init_boundary`:
    `0` when `--begin 0` (t=0 init is baseline), `t0 - 1` when `--begin > 0` (strictly-before-window baseline)
  - skip threshold uses `t <= init_boundary` instead of `t <= t0`

## Behavior Change

Before `v1.3.10`:

- `summary` treated `t == --begin` events as already absorbed into the
  baseline snapshot when `--begin > 0`
- those begin-boundary events did not contribute to changes / rise_count /
  fall_count / first_at

In `v1.3.10`:

- `summary` properly counts value_changes exactly at `--begin` as part of
  the inclusive query window when `--begin > 0`, fixing the boundary
  black-hole where transitions at the window edge were absorbed into
  baseline and silently dropped from changes / rise_count / fall_count /
  first_at.
- `summary --begin 0` is unchanged from `v1.3.9`: t=0 initialization
  events establish baseline state and are not counted as changes. This
  preserves the "static / active" classification on real-world VCDs
  where many signals only carry $dumpvars init dumps.

The `kind` classification rule is unchanged from `v1.3.9`:
- `active` = at least one value_change observed strictly after t=0 in
  the window
- `static` = known initial value, no value_changes in window
- `undefined` = selected but never observed

Compatibility impact:

- scalar and vector `summary` results can change at the window boundary
  when `--begin > 0`

## Regression Focus For This Version

The targeted regressions added for `v1.3.10` verify:

- begin-boundary scalar rise is counted in `changes` and `rise_count` (`--begin 10ns`)
- begin-boundary scalar fall is counted in `changes` and `fall_count` (`--begin 10ns`)
- `summary --begin 0` excludes `$dumpvars` initialization from changes
- begin-boundary vector transitions set expected `first_at`, `last`, and `unique`
- `dump` and `summary` agree that begin-boundary events are in-range
- an init-only signal at `--begin 0` is `static` with `changes=0`

Full suite coverage remains in place for:

- `info --json`
- `list --json --filter`
- `dump --begin --end --filter`
- `summary --begin --end --filter`
- `snapshot --at --filter`
- `compare --at --filter`
- `search` primary interface for this version
- `--version`
- `--limit` and `--verbose` on representative streaming/report commands
- literal bus-range glob regression (`*data[7:0]`)
- escaped identifier scope regression
- `_normalize_filter_patterns` rejects non-sequence input
- compare rejects reversed time range
- search on empty VCD reports input error
- info/dump agreement on malformed vector/real/port tokens
- valid multichar port parsing consistency in info and dump
