## Release Headline

Harden input validation and error reporting

## Feature / Fix Summary

- Reject non-str/non-sequence input in `_normalize_filter_patterns` with a clear error instead of `AttributeError`.
- Report empty VCD data sections as an input/data issue rather than silently returning "no match" from `search`.
- Validate `compare` time ordering (must be T1 <= T2) with a clear error message.
- Add inline documentation note explaining that search requires at least one value_change in the VCD.

## CLI Surface Changes

No CLI subcommands were added or removed in this version.

## Code-Level Change Shapes

Modified helpers:

- `_normalize_filter_patterns`: added `isinstance` guard for str/list/tuple/set/sequence, raises `_FilterParseError` on unexpected types.
- `_search_end_time`: raises `_ConditionParseError` when `scan_time_range()` returns None (empty data section).
- `cmd_compare`: added `tb < ta` ordering check with `_TimeParseError`.
