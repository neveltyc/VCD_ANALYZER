## Release Headline

Eliminate duplicated value-change parsing in data scanning paths

## Feature / Fix Summary

- Extract `_is_structural_token()` and `_consume_value_change()` as shared VCDParser methods.
- Replace the two independent copies of b/r/p value-change parsing in `iter_events()` and `scan_time_range()` with calls to the shared helper.
- Guarantee that `info`'s reported time range and `dump`/`search` parsing behaviour stay aligned when malformed-token rules evolve.

## CLI Surface Changes

No CLI subcommands were added or removed in this version.

## Code-Level Change Shapes

Added VCDParser methods:

- `_is_structural_token(tok)`: single canonical location for the "#<digits> vs symbol" disambiguation.
- `_consume_value_change(tok, next_token, pushback)`: single canonical validation path for scalar/vector/real/port value_change tokens.

Removed:

- Local `_looks_structural` closure in `iter_events()`.
- Local `_is_struct` closure in `scan_time_range()`.
- ~100 lines of duplicated b/r/p parsing across both methods.
